# シナリオ概要

本システムは、オンプレミス環境からAWS上のデータベース（DB）に格納された機密情報へアクセスするアプリケーションを構築します。

- 機密情報が保存されているDBは外部公開されておらず、AWS VPC内のEC2上で稼働するアプリケーションからのみアクセス可能とします。
- オンプレミス環境とAWS間はVPN接続によりセキュアな通信を確保し、機密情報への安全なアクセスを実現します。

## セキュリティリスクへの対応

近年、VPN接続のセキュリティホールを突かれたセキュリティ事故が多発しています。
そのため、VPNの脆弱性を悪用した外部からの不正アクセスが発生した場合も考慮します。

- GuardDutyによる侵入検知を導入し、異常なアクセスを早期に検知します。
- セキュリティグループを活用し、不正アクセスに対する防御機能を強化します。

# 全体構成
AWS側のprod-VPCと疑似オンプレミス環境のonpre-VPC
の2つを用意しました。

両VPC間はVPN接続によりセキュアに接続されています。

prod-VPCには、WebServerとDBServerを配置しています。
onpre-VPCには、RouterとUser操作サーバーを配置しました。

DBServerはプライベートに配置し、外部からアクセス出来ないようにしました。

基本設計
AWS側VPCと疑似オンプレVPCを用意
疑似オンプレからAWS側の機密情報にVPN経由でアクセスする

- CIDR：10.0.0.0/16
    - パブリックサブネット

     - Name: prod-hcsa-public-2（10.0.2.0/24, us-east-2b）
    - プライベートサブネット
     - Name: prod-hcsa-private-1（10.0.11.0/24, us-east-2a）
     - Name: prod-hcsa-private-2（10.0.12.0/24, us-east-2b）
 - ACL名：prod-hcsa-vpc-acl
 - セキュリティグループ名：prod-hcsa-vpc-sg
 - ルートテーブル名：prod-hcsa-vpc-rtb
 - インターネットゲートウェイ名：prod-hcsa-vpc-igw
 - 仮想プライベートゲートウェイ名：prod-hcsa-vpc-vpgw

#設計2 DB設計
基本設計はMYSQLを使って、プライベートアクセスだけ決定。

詳細設計（例）

【AWS側】
-機密情報保持DBサーバー
 --Name：prod-hcsa-vpc-rds
 --engin：Mysql
 --instance Type:db.t4.micro
 --strage：gp2 20gb
 --vpc：prod-hcsa-vpc
 --AZ：us-east-2a
 --subnet：prod-hcsa-private-1
 --security group：prod-hcsa-vpc-rds-sg
 --AllowPublicAccess：false
 --interface：default private us-east-2a
 --authentication:IAM認証
 --masteruser:admin pass:

スキーマ設計
-DB名：hcsa_db
-テーブル名：users
 --id:INT PRIMARY KEY AUTO_INCREMENT
 --name:VARCHAR(255) NOT NULL
 --email:VARCHAR(255) NOT NULL UNIQUE
 --password:VARCHAR(255) NOT NULL
 --created_at:DATETIME DEFAULT CURRENT_TIMESTAMP
 --updated_at:DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP

#設計5 セキュリティ設計
基本設計
GuardDutyで攻撃を検知し、
EventBridgeでイベントLambdaに転送
Lambdaでイベントから攻撃者のIPアドレスを取得
DenyIPセットに動的登録

詳細設計（例）
【GuardDuty】
region: us-east-2

【EventBridge】
rulename: hcsa-eventbridge-guardduty-finding-to-lambda
region: us-east-2
eventbusName: default
eventtargets:
	- block_attacker_ip_port (Lambda)
filterrule:
	- UnauthorizedAccess（不正アクセス）
	- Recon（偵察行為）
	- Backdoor（バックドア検知）
	- CryptoCurrency（仮想通貨マイニング）
	- Impact（破壊・改ざん行為）
	- Persistence（永続化の試み）
	- PenTest（ペネトレーションテスト検知）
	- Behavior（異常なふるまい）
	- Trojan（マルウェア検知）

詳細設計
【hcsa-ec2-appserver-role】
--必要権限
SSM Sessionマネージャーでアクセス出来る
　→ AmazonSSMManagedInstanceCore（arn:aws:iam::aws:policy/AmazonSSMManagedInstanceCore）
SSHでアクセス出来る
　→ （キーペア管理、IAM権限不要）
RDSにIAM認証で入れる
　→ AmazonRDSFullAccess または rds-db:connect
cloudwatchにログを書き込める
　→ CloudWatchAgentServerPolicy（arn:aws:iam::aws:policy/CloudWatchAgentServerPolicy）
RDSのデータを参照できる、編集できる、追加できる
　→ rds-db:connect, rds:DescribeDBInstances, rds:ExecuteStatement
