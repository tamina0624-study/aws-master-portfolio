# 📘 Kubernetes学習まとめ（図解付き）

---

# 1. コンテナ技術の全体像

## ■ コンテナとは
アプリケーションと実行環境をまとめてパッケージ化したもの。

---

## ■ 役割分担

```
アプリ
 ↓
Docker（コンテナ化）
 ↓
Kubernetes（運用）
```

- Docker：コンテナを作る・動かす
- Kubernetes：コンテナを管理・運用する

---

# 2. Kubernetesの本質

## ■ 一言でいうと
 コンテナの自動運用システム

---

## ■ できること

```
[ユーザーアクセス増]
        ↓
  自動スケール
        ↓
[コンテナ増加]
```

- 自動復旧
- 自動スケーリング
- 無停止デプロイ
- 負荷分散

---

# 3. 基本構造（超重要）

```
Cluster
 ├ Node（サーバ）
 │   └ Pod（最小単位）
 │       └ Container
```

---

## ■ リソース関係

```
Deployment
   ↓ 管理
 Pod（複数）
   ↓
Container
```

---

## ■ 通信の流れ

```
外部アクセス
     ↓
 Service
     ↓
   Pod
```

---

# 4. Kubernetesの動作原理

## ■ 宣言型（Declarative）

```
replicas: 3
```

 「3個にしたい」と書くだけでOK

---

## ■ 内部の動き

```
理想状態（3個）
   ↓
現在状態（2個）
   ↓
差分検知
   ↓
自動で1個起動
```

 常に理想状態に修正される

---

# 5. Dockerとの関係

```
Docker → コンテナ作成
Kubernetes → コンテナ管理
```

---

## ■ 比較

| 機能 | Docker | Kubernetes |
|------|--------|------------|
| コンテナ作成 | ○ | × |
| 実行 | ○ | ○ |
| スケール | × | ○ |
| 自動復旧 | × | ○ |

---

# 6. AWSとの関係

## ■ 構成イメージ

```
[開発]
  ↓
Docker build
  ↓
ECR（イメージ保存）
  ↓
EKS（Kubernetes）
  ↓
EC2上で実行
```

---

## ■ 対応関係

| Kubernetes | AWS |
|------------|-----|
| Node | EC2 |
| Service | ALB |
| Volume | EBS |

---

# 7. ECSとの違い（重要）

## ■ 全体イメージ

```
ECS
 → AWS専用（シンプル）

Kubernetes
 → 汎用（高機能・複雑）
```

---

## ■ 設計思想

```
ECS：AWS最適化
Kubernetes：どこでも動く
```

---

## ■ 対応関係

| Kubernetes | ECS |
|------------|-----|
| Pod | Task |
| Deployment | Service |
| Node | EC2 |

---

# 8. メリット・デメリット

## ■ メリット

```
✔ 自動復旧
✔ 自動スケール
✔ 高可用性
✔ 柔軟性
```

---

## ■ デメリット

