📝 物理・論理コンポーネント構成の説明

💻 ローカルPC (操作元)

  • Terraform: terraform apply を実行し、AWSのAPIを叩いてVPC、EC2、EKSクラスターなどの「物理的・インフラ的」なリソースを自動構築します。
  • kubectl: 構築されたKubernetesのAPIサーバーと通信し、Nginxなどの「論理的」なリソース（Pod、Deployment、Serviceなど）を展開・管理します。

☁️ セルフマネージド構成 (K3s EC2)

  • 物理配置: パブリックサブネット1に配置された1台のEC2インスタンス（t3.small）です。
  • K8s内部構造: 1台の中にコントロールプレーン（APIサーバー等）とワーカーノード機能（kubelet等）が同居しています。その内部に Pod (Nginx) がコンテナとして稼働しています。

🧠 マネージド構成 (Amazon EKS)

  • EKS Control Plane (マスターノード): AWSの管理下（見えないVPC領域）に配置されたコントロールプレーンです。APIリクエストを受け取る API Server、クラスタ状態を保存する etcd などの中核機能が動いています。
  • EKS Worker Node (ワーカーノード): 私たちのVPC（サブネット1と2）に配置された実体のEC2インスタンス（t3.medium）です。
  • K8s内部構造: ワーカーノード内では kubelet（ノードの管理者）と kube-proxy（ネットワーク管理）が常駐し、コントロールプレーンからの指示を受けて Pod (Nginx等) を起動・維持しています。
