# AWS SSO（IAM Identity Center）と最小権限ライフサイクル管理

## 目的
- AWS IAM Identity Center（旧SSO）をTerraformでコード化し、エンタープライズ基準のID・アクセス管理を実現する
- ロールごとの最小権限設計、ユーザー・グループ管理、権限ライフサイクル（入社・異動・退職）自動化の実践
- トラブルや落とし穴も含めた実務的な知見の蓄積

## 学習・実装ステップ
1. SSO（IAM Identity Center）基本リソースのTerraform定義
2. Permission Set（ロール）の設計・実装
3. ユーザー・グループの定義と割り当て
4. 権限ライフサイクル（入社・異動・退職）のシナリオ例
5. トラブル・落とし穴の検証メモ

## 目標アウトプット
- Terraformコード一式
- 設計・運用ノウハウのドキュメント
- 想定トラブルとその解決策
